Docker

A docker programokat futtat a container-ekben (mint pl. nginx, Apache, MySQL vagy Wordpress/Drupal). Ezeket az operációs rendszeredtől külön futtathatod. Szóval ahelyett hogy felépítenél egy alkalmazást egy gépen és megpróbálnád ugyanazt másolni csak simán futattod a container-t és ugyanaz lesz Mac-en, Linuxon és Windowson is. Ha működik a te gépeden, működik a szerveren és az összes gépen. Dockerrel több projekt is elkülönítve maradhat és könnyebb kezdeni valaki más projektjével (nem kell feltelepíteni az összes eszközt/dependenciest). Egy Virtuális géphez képest kevesebb erőforrást használ, kevesebb helyet foglal és gyorsabban indul.

Image és container

  • Az image olyan mint egy tervrajz, a container olyan mint egy ház
  • A dockerfile meghatározza hogy milyen lesz az image (tervrajz)
  • Nem építhetsz házat tervrajz nélkül
  • A ház tervrajzát DockerHubról tudod megszerezni vagy Dockerfileból
  • Elindítani elsőre egy containert olyan mint felépíteni egy házat és kinyitni az ajtaját
  • Megállítani egy containert olyan mint bezárni az ajtót
  • Törölni egy containert olyan mint lebontani a házat

Hasznos parancsok

docker container run -d -p 80:80 nginx

  1. Keresi az nginx image-t az image gyorsítótárban (cache)
  2. Ha nincs a gyorsítótárban, felkeresi az image repo-t Dockerhubról
  3. Leszedi a legújabb verziót, elmenti a gyorsítótárba
  4. Elindítja egy új container-ben
  5. a 80 as port a host a container port 80-on

docker container ls -a

megmutatja az összes containert (ami fut azt is)

docker container stop [ID]

megállítja a containert

docker container rm -f [ID]

Force törli a containert (így nem kell megállítani)

docker image ls

megmutatja az összes imaget ami le van szedve

docker pull [image]

Leszedi az adott imaget

docker image rm [image]

Törli az adott imaget
